# NixCore OS - Обзор архитектуры

## Архитектура системы

NixCore - это современная 64-битная операционная система с монолитной архитектурой ядра. Система спроектирована с учетом модульности, сохраняя при этом высокую производительность за счет прямого доступа к оборудованию.

## Управление памятью

### Виртуальная память

NixCore использует 4-уровневую страничную организацию для управления виртуальной памятью:

- **PML4** (Page Map Level 4) - Верхний уровень
- **PDP** (Page Directory Pointer) - Второй уровень
- **PD** (Page Directory) - Третий уровень
- **PT** (Page Table) - Нижний уровень

Каждая страница имеет размер 4КБ. Система поддерживает:
- Подкачку по требованию
- Copy-on-Write (COW) для fork()
- Свопинг на диск
- Файлы, отображаемые в память

### Физическая память

Физическая память управляется с помощью битового аллокатора:
- Каждый бит представляет одну страницу 4КБ
- Быстрое выделение/освобождение
- Отслеживание свободных и занятых страниц
- Поддержка зон памяти

### Управление кучей

Ядро и пользовательское пространство используют отдельные аллокаторы кучи:
- **Куча ядра**: Простой bump-аллокатор с объединением
- **Пользовательская куча**: malloc/free с разделением и слиянием блоков

## Управление процессами

### Планировщик

Вытесняющий планировщик с 5 приоритетами:
- **Приоритет 0**: Реального времени (наивысший)
- **Приоритет 1**: Высокий
- **Приоритет 2**: Нормальный
- **Приоритет 3**: Низкий
- **Приоритет 4**: Простой (наименьший)

Циклическое планирование внутри каждого уровня приоритета с квантами времени 10мс.

### Переключение контекста

Полное сохранение/восстановление состояния CPU:
- Регистры общего назначения (RAX, RBX, RCX, RDX, RSI, RDI, RBP, RSP)
- Сегментные регистры (CS, DS, ES, FS, GS, SS)
- Управляющие регистры (CR0, CR2, CR3, CR4)
- Состояние FPU (FXSAVE/FXRSTOR)
- Указатель инструкций (RIP)
- Регистр флагов (RFLAGS)

## Файловая система

### Виртуальная файловая система (VFS)

Единый интерфейс для всех файловых систем:

```c
struct filesystem_operations {
    int (*mount)(filesystem_t *fs, const char *device);
    inode_t *(*get_root)(filesystem_t *fs);
};

struct inode_operations {
    inode_t *(*lookup)(inode_t *dir, const char *name);
};

struct file_operations {
    ssize_t (*read)(inode_t *inode, void *buf, size_t count, uint64_t offset);
    ssize_t (*write)(inode_t *inode, const void *buf, size_t count, uint64_t offset);
};
```

### Поддерживаемые файловые системы

**FAT32**:
- Разбор загрузочного сектора
- Кэширование таблицы FAT
- Следование цепочкам кластеров
- Поиск записей каталога
- Поддержка длинных имен файлов (планируется)

**ext2**:
- Суперблок и дескрипторы групп
- Управление таблицей inode
- Прямые/косвенные/двойные/тройные косвенные блоки
- Разбор записей каталога
- Символические ссылки (планируется)

**devfs**: Файлы устройств (/dev/null, /dev/zero, /dev/random, /dev/sda*)
**procfs**: Информация о процессах (/proc/cpuinfo, /proc/meminfo, и т.д.)
**sysfs**: Системная информация и управление

## Сетевой стек

### Архитектура уровней

```
Прикладной уровень   │ HTTP, DNS, NTP
Транспортный уровень │ TCP, UDP
Сетевой уровень      │ IP, ICMP
Канальный уровень    │ Ethernet, ARP
Физический уровень   │ Драйвер E1000
```

### API сокетов

POSIX-совместимый интерфейс сокетов:

```c
int socket(int domain, int type, int protocol);
int bind(int sockfd, const sockaddr_t *addr, uint32_t addrlen);
int listen(int sockfd, int backlog);
int connect(int sockfd, const sockaddr_t *addr, uint32_t addrlen);
ssize_t send(int sockfd, const void *buf, size_t len, int flags);
ssize_t recv(int sockfd, void *buf, size_t len, int flags);
```

### Конечный автомат TCP

```
CLOSED → LISTEN → SYN_RECEIVED → ESTABLISHED → FIN_WAIT_1 → 
FIN_WAIT_2 → TIME_WAIT → CLOSED

CLOSED → SYN_SENT → ESTABLISHED → FIN_WAIT_1 → FIN_WAIT_2 → 
TIME_WAIT → CLOSED
```

## Драйверы устройств

### AHCI (SATA)

- Сканирование шины PCI
- Инициализация портов
- Настройка списка команд и FIS
- DMA-передачи
- Команды READ/WRITE DMA EXT

### NVMe (PCIe SSD)

- Настройка очереди администратора
- Пары очередей ввода-вывода
- Очереди отправки/завершения
- Команды READ/WRITE
- Обработка прерываний

### E1000 (Intel Gigabit Ethernet)

- Доступ к регистрам MMIO
- Кольца дескрипторов RX/TX
- Передача пакетов
- Прием пакетов
- Чтение MAC-адреса из EEPROM

### USB стек

**XHCI (USB 3.0)**:
- Кольцо команд
- Кольцо событий
- Кольцо передачи
- Управление слотами

**EHCI (USB 2.0)**:
- Асинхронное расписание
- Периодическое расписание
- Головы очередей
- Дескрипторы передачи

**UHCI (USB 1.1)**:
- Список кадров
- Головы очередей
- Дескрипторы передачи

## Процесс загрузки

1. **UEFI Firmware** - Загружает загрузчик
2. **Загрузчик** - Настраивает GOP, загружает ядро
3. **Вход в ядро** - Инициализирует GDT, IDT, страничную организацию
4. **Инициализация памяти** - Настройка PMM и VMM
5. **Инициализация драйверов** - AHCI, NVMe, Сеть, USB
6. **Инициализация VFS** - Монтирование корневой ФС
7. **Запуск планировщика** - Создание процесса init
8. **Процесс Init** - Монтирование других ФС, запуск служб
9. **Рабочий стол** - Запуск GUI и оболочки

## Соображения производительности

- **Сеть без копирования**: Прямой DMA в пользовательские буферы
- **Кэш страниц**: Кэширование часто используемых блоков диска
- **Ленивое выделение**: Выделение памяти при первом доступе
- **COW fork**: Совместное использование страниц до записи
- **Объединение прерываний**: Пакетная обработка прерываний

## Функции безопасности

- **ASLR**: Рандомизация размещения адресного пространства
- **NX бит**: Страницы без выполнения
- **Разделение пользователь/ядро**: Кольцо 0/Кольцо 3
- **Безопасность на основе возможностей** (планируется)
- **Песочница** (планируется)
